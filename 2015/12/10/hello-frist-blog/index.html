hexo 总结

hexo
  • hexo
  • 随笔

摸索了一天一夜了，终于搞定了 mac + github + hexo。

搭建hexo博客之前 你应该具备什么？

你需要一颗好奇的新，肯花时间就行。博主就是零基础，什么node.js，之前就没听过。所以零基础的你一定可以的。

需要的工具
一台mac,一个github账户

什么是hexo
出自tommy351之手，hexo是一个基于Node.js的静态博客程序，其编译速度之快，让我很是惊讶。hexo生成的静态网页可以直接放到GitHub Pages，BAE，SAE等平台上。
之前弄过Wordpress,最初的想法是用它来学习HTML，就注册了域名，还花了几十块钱。结果，一点html也没有用到。结果就不了了之，几十块钱也打水漂了。是不是我太傻不会弄啊？
后来看到了hexo，又激起了我当初的那个做博客的梦，毕竟几十块钱不能白花啊。
下面总结建站过程中的一些总结。也当是笔记了，以后找起来也方便。

关于 CNAME
在电脑上的hexo所建立的网站路径下有个source文件夹，就把CNAME放在这里就可以了。每次 hexo generate时，会把CNAME拷贝到./punlic文件夹下。而每次hexo deploy时，又会把./public文件夹下的内容同步到github中。

关于 插入优酷视频
打开想链接的优酷上的视频，点击视频下面的分享，会有html链接。
<embed src="http://player.youku.com/player.php/... ></embed>
把这个链接直接复制到.md 文件中即可。亲测。
