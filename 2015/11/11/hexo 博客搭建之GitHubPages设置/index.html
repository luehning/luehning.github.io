博客搭建之GitHubPages设置

发表于 2015-11-11   |   分类于 网站搭建   |   | 阅读量 次

文章内容：

  1. GitHubPages的创建
  2. CNAME文件的创建
  3. DNS设置

  前面说了在本地搭建hexo环境，现在要做的是把本地的博客，托管到GitHub上。为什么呢？GitHub为博主们免费搞了500Mb的空间，那么我们就不用在买空间了不是么？

GitHubPages的创建

  说到这里还不快快申请个GitHub账号。哈哈，申请完成后，就开创一个repository吧，所谓repository,就是一个空间。

  new repository 后就会进入下面页面:

  name中填写 你的名字.GitHub.io，然后选择public，可以选择创建README或者不创建。点击 Create repository

申请完成后是这样的：

点击后，进入

在浏览器输入创建的地址就可以打开我们建的页面啦，但是还没完，如果给这个地址添加一条记录，通过一个域名来访问岂不是完美。

新建CNAME文件

  上图中，点击 New File，创建名为CNAME，内容为你的域名（唯一），不可以写多个域名。
  创建完成后，下载到博客的Blog/source文件夹下。之前说过，hexo g编译文件时会把source文件夹下的文件编译并拷贝到public文件夹下。而通过hexo d命令又会把public文件夹下的文件同步到GitHub中，也就是我创建的luehning.github.io中。
  这样，在GitHub这边的设置就全部做完了。但是还没完，

DNS设置

我的域名是在godaddy上买的。登陆后，选择Domains，直接点击Domains即可

选择Manage DNS
