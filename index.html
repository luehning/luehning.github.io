博客搭建之hexo环境

发表于 2015-12-10   |   分类于 hexo
摸索了两天一夜了，终于搞定了 mac + github + hexo。。。
我想说hexo很值得做。markdown语法也很有意思，一定要get。
第二次安装hexo搭建博客，原因是格式化了硬盘，重新做了系统。
本以为备份了博客就可以简单的拷贝然后直接使用。
事实证明这样是行不通的。

# 搭建hexo博客之前 你应该具备什么？
你需要一颗好奇的新，肯花时间就行
一台mac,一个github账户

#什么是hexo

一个快速、简洁且高效的博客框架。
出自tommy351之手，hexo是一个基于Node.js的静态博客程序，其编译速度之快，让我很是惊讶。
hexo生成的静态网页可以直接放到GitHub Pages，BAE，SAE等平台上。
之前弄过Wordpress,最初的想法是用它来学习HTML，就注册了域名，还花了几十块钱。
结果，一点html也没有用到。结果就不了了之，几十块钱也打水漂了。是不是我太傻不会弄啊？
后来看到了hexo，又激起了我当初的那个做博客的梦，毕竟几十块钱不能白花啊
下面总结建站过程中的一些总结。也当是笔记了，以后找起来也方便。

安装HomeBrew

homebrew是一个应用管理器，安装它纯粹是为了以后的安装需要。
它在以后写博客时用到的几率为零。
打开终端，输入下面命令，执行如下图：

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"



下面是brew 的命令，安装NodeJS时会用 install 命令。

安装NodeJS

brew install node

安装Hexo

安装hexo分四步
1.终端中使用cd 命令进入要保持博客的文件夹，（自己新建一个）我建立个文件夹为blog。
2.输入命令npm install hexo -g

3.实例化hexo,使用命令hexo init

4.安装图上提示，最后 一步执行 npm install

安装GIT

brew install git

获取SSH 秘钥

ssh 秘钥的好处是每次上传到GitHub都不需要再输入密码了。

1.执行命令ssh-keygen -t rsa -C 如图后面跟自己的邮箱

2.git config --global user.name 邮箱名 注 邮箱名为 @ 前面部分
3.git config --global user.email "邮箱名@163.com"

到此秘钥已经生成了，我们最终要的是秘钥，该秘钥存储在.pub文件里。
cd ~/.ssh/ 秘钥就悄悄保存在这里。这就好办了，使用ls查看吧。

4.复制秘钥信息 cat id_rsa.pub



# GitHub中设置SSH秘钥
设置秘钥要登陆GitHub账户，登陆后点这里。

点击Add SSH Key


Title随便写，不过建议使用 blog 什么之类的，方便以后使用。

key呢？
就把之前cat 后得到的秘钥全部复制(包括ssh-rss 和 邮箱)，
粘贴进这里。然后ADD Key。
如何知道自己是不是设置成功了，下面我们验证一下。


输入第一条命令 ssh -T git@github.com，
会提示是否继续，输入yes,显示入上图下面所示，
那么恭喜你已经设置成功，环境搭建已经基本结束了。

Hexo 设置

下图是我的博客的根目录的样子，
_config.xml是博客的设置文件，
themes里面是主题，我目前使用的next主题，曾经使用过gelee主题。
所有的文章都会放在source里。
编译完了之后会把source里的文件放进public文件夹里。
GitHub备份的也正是public文件夹。


## 修改_config.xml文件
deploy:
  type: git
  repo: git@github.com:xxxx/xxxx.github.io.git
 branch: master

github设置请参考这里
到这里 已经可以在本地查看博客状态了。

hexo server

在浏览器中输入 http://0.0.0.0:4000/ 来访问本地hexo服务器。

# 总结笔记

下面总结建站过程中的一些总结。也当是笔记了，以后找起来也方便。

关于 CNAME

在电脑上的hexo所建立的网站路径下有个source文件夹，就把CNAME放在这里就可以了。每次 hexo generate时，会把CNAME拷贝到./punlic文件夹下。而每次hexo deploy时，又会把./public文件夹下的内容同步到github中。
